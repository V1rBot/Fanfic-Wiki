Данные хранятся в виде стандартизированных полей (Data Types) в карточке каждого сотрудника, агента или объекта, проявляющего личность.

#### **1. БАЗОВЫЙ СТАТУС (Core Status)**
*   **Тип данных:** Классификатор (Classifier) - выбор из списка.
*   **Возможные значения:**
    *   `Активен` (Жив, дееспособен, находится при исполнении обязанностей).
    *   `Неактивен` (Жив, но не находится на duty: отпуск, отстранение, арест).
    *   `Потерян` (Пропал без вести, статус неизвестен. Для объектов - вышел из-под контроля).
    *   `Умерлен` (Погиб, тело идентифицировано).
    *   `Умерщвлен` (Убит с подтверждением).
    *   `Неизвестно` (Статус не может быть установлен: амнезия, трансформация, пр.).
    *   `Аннулирован` (Ликвидирован по приказу Фонда).
    *   `Эксгумирован` (Для объектов, чьё состояние "смерти" обратимо или неопределенно).

#### **2. БИОЛОГИЧЕСКИЕ / ФИЗИОЛОГИЧЕСКИЕ ДАННЫЕ (Biological/Physiological Data)**
*   **Тип данных:** Набор классификаторов и текстовых полей.
*   **Поля:**
    *   `Биологический пол`: Мужской, Женский, Неопределенный, Неприменимо (N/A), Изменчивый, Аномальный.
    *   `Вид / Раса`: *(Важное поле для сортировки)*
        *   **Для людей:** Homo Sapiens, Генномодифицированный Homo Sapiens, Парачеловек.
        *   **Для SCP-объектов:** Выбор из списка + текстовое поле для уточнения.
            *   `Неизвестного происхождения`
            *   `Гуманоид` (e.g., SCP-049, SCP-096)
            *   `Фауна` (e.g., SCP-939, SCP-682)
            *   `Флора` (e.g., SCP-407)
            *   `Механический/Искусственный` (e.g., SCP-076-2, многие объекты Аппарата)
            *   `Эфирный/Бесплотный` (e.g., SCP-966, SCP-303)
            *   `Паразитический/Симбиотический` (e.g., SCP-516)
            *   `Когнитивный/Меметический` (e.g., SCP-2521, SCP-3125)
            *   `Божественное/Сверхъестественное существо`
    *   `Дата рождения/Создания`: Календарная дата или "Неизвестно".
    *   `Внешний возраст`: Приблизительный визуальный возраст.
    *   `Рост / Вес`: Числовые значения или "Изменчивый", "Неприменимо".

#### **3. СЛУЖЕБНЫЕ ДАННЫЕ (Service Data)**
*   **Тип данных:** Ссылки и классификаторы.
*   **Поля:**
    *   `Идентификатор сотрудника/объекта`: (e.g., "Dr. Alto Clef", "SCP-073", "MTF E-11 Командир").
    *   `Уровень допуска`: 0 (Только самые базовые данные), 1 (Общий персонал), 2 (Безопасность), 3 (Старшие исследователи), 4 (Администрация объекта), 5 (O5 Совет).
    *   `Должность/Роль`: Исследователь, Агент, Специалист по содержанию, Директор, Объект содержания, D-класс, Командующий МОГ.
    *   `Подразделение`: Ссылка на отдел (e.g., "Исследовательский отдел Site-19", "MTF Zeta-9"). Для D-класса - "Персонал класса D".
    *   `Место приписки/содержания`: Ссылка на объект Фонда (e.g., "Site-17", "Armed Area-14").

#### **4. ОСОБЫЕ ПРИМЕТЫ / АНОМАЛЬНЫЕ СВОЙСТВА (Distinguishing Features / Anomalous Properties)**
*   **Тип данных:** Текстовый блок с тегами.
*   **Описание:** Поле для описания уникальных физических черт, способностей, ограничений или аномальных эффектов.
*   **Примеры тегов:** `[РЕГЕНЕРАЦИЯ]`, `[МЕМЕТИЧЕСКАЯ УГРОЗА]`, `[НЕВИДИМОСТЬ]`, `[УМСТВЕННОЕ ВЛИЯНИЕ]`, `[ФИЗИЧЕСКАЯ НЕУЯЗВИМОСТЬ]`, `[ОБЛАДАЕТ СВЯЗЬЮ С SCP-XXXX]`.
